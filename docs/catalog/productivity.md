# Productivity, Publishing & Communication

A catalog of self-hostable tools for search, notifications, feeds, publishing, social networking, messaging, business operations, collaboration, note-taking, and knowledge management. Each entry links to the upstream project with a short factual description.

[← Catalog index](./README.md) · [← Handbook home](../../README.md)

## Search

Search engines, metasearch frontends, and full-text indexing backends you can run yourself.

- [Meilisearch](https://github.com/meilisearch/meilisearch) — fast search engine that integrates into apps, websites, and workflows.
- [Shodan](https://www.shodan.io/) — search engine for Internet-connected (IoT) devices.
- [Whoogle Search](https://github.com/benbusby/whoogle-search) — self-hosted, ad-free, privacy-respecting metasearch engine.
- [SearX](https://github.com/searx/searx) — privacy-respecting, hackable metasearch engine.
- [SearXNG](https://github.com/searxng/searxng) — metasearch engine that aggregates results from various search services and databases.
- [Sonic](https://github.com/valeriansaliou/sonic) — lightweight, schema-less search backend and Elasticsearch alternative that runs on a few MBs of RAM.
- [Zinc](https://github.com/zinclabs/zinc) — full-text indexing search engine and lightweight alternative to Elasticsearch.
- [Cylect.io](https://cylect.io/) — searching tool for finding information across websites, search engines, and data collectors.
- [Lyra](https://docs.lyrasearch.io/) — in-memory, typo-tolerant, full-text search engine written in TypeScript.
- [Hugo Lyra](https://github.com/paolomainardi/hugo-lyra) — TypeScript module for creating LyraSearch indexes for static Hugo sites, with server and client libraries.
- [Typesense](https://github.com/typesense/typesense) — typo-tolerant search engine for building search experiences.
- [Tantivy](https://github.com/quickwit-oss/tantivy) — full-text search engine library inspired by Apache Lucene and written in Rust.
- [Toshi](https://github.com/toshi-search/Toshi) — full-text search engine similar to Elasticsearch, built on Tantivy.
- [FlexSearch](https://github.com/nextapps-de/flexsearch) — full-text search library for the browser and Node.js.
- [fd](https://github.com/sharkdp/fd) — fast, user-friendly alternative to find for locating entries in the filesystem.
- [k8s at home search](https://nanne.dev/k8s-at-home-search/#/) — indexes Flux HelmReleases from GitHub repositories tagged with the `k8s-at-home` topic.
- [OpenFind](https://open.getfind.app/) — app to find text in your photo library, running fully offline.

## Notifications

Services and libraries for delivering push and pub-sub notifications.

- [Apprise](https://github.com/caronc/apprise) — sends notifications to many services including Telegram, Discord, Slack, Amazon SNS, and Gotify.
- [ntfy](https://ntfy.sh/) — HTTP-based pub-sub notification service for sending notifications to phone or desktop via scripts; open source and self-hostable.
- [Countly](https://github.com/Countly/countly-server) — product analytics solution that tracks product performance and customer behavior across mobile, web, and desktop, with privacy by design.
- [notifiers](https://github.com/liiight/notifiers) — wrapper for a variety of third-party providers and built-in ones like SMTP, aimed at sending notifications.
- [Pushover](https://pushover.net/) — real-time notifications on Android, Android Wear, iPhone, iPad, Apple Watch, and desktop.
- [Simplepush](https://simplepush.io/) — sends end-to-end encrypted push notifications to Android and iPhone.
- [UnifiedPush](https://unifiedpush.org/) — free and open specifications and tools that let the user choose how push notifications are delivered.

## RSS

Feed readers, aggregators, and generators for RSS, Atom, and JSON feeds.

- [RSS Guard](https://github.com/martinrotter/rssguard) — RSS/ATOM/JSON feed reader for Windows, Linux, BSD, OS/2, and macOS that works with online services including Feedly, Gmail, Google Reader API services (Bazqux, FreshRSS, Inoreader, Miniflux, Reedah, The Old Reader), Nextcloud News, and Tiny Tiny RSS.
- [Feedly](https://feedly.com/) — RSS tool to organize topics and trends, with integrations for Facebook, Twitter, Evernote, Buffer, OneNote, Pinterest, LinkedIn, IFTTT, and Zapier.
- [FreshRSS](https://www.freshrss.org/) — self-hosted, lightweight, customizable RSS and Atom feed aggregator.
- [ArchiveBox](https://archivebox.io/) — self-hosted internet archiving tool that saves URLs, browser history, bookmarks, and more as HTML, JS, PDFs, and media for offline viewing.
- [RSSHub](https://github.com/DIYgod/RSSHub) — open source, extensible RSS feed generator capable of generating feeds from many sources.
- [Miniflux V2](https://github.com/miniflux/v2) — minimalist and opinionated feed reader.

## Nostr

Nostr (Notes and Other Stuff Transmitted by Relays) is a protocol that aims to create a censorship-resistant global social network, based on simple JSON event objects and standard elliptic-curve cryptography for keys and signing. The links below cover the protocol docs, relays, clients, libraries, and tooling.

### Protocol & guides

- [nostr](https://github.com/nostr-protocol/nostr) — protocol overview and FAQ.
- [NIPs](https://github.com/nostr-protocol/nips) — Nostr Implementation Possibilities describing the protocol in technical detail.
- [nostr, a basic tour](https://github.com/rajarshimaitra/rust-nostr/blob/main/VISION.md) — an introduction to Nostr.
- [UseNostr](https://usenostr.org) — guide to how Nostr works and what it can do.
- [nostr.how](https://nostr.how) — quick-start to onboard desktop users with Alby and Astral.
- [nostr.guide](https://nostr.guide) — guide to all things Nostr.
- [nostr address book](https://github.com/aitechguy/nostr-address-book) — directory of Twitter user accounts and their Nostr addresses.
